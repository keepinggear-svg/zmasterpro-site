# 支先森制造官网内容维护指南

## 1. 怎么修改案例

案例数据在：

```text
content/cases.json
```

每个案例是一组 JSON 数据：

```json
{
  "title": "街舞机构夏季校服",
  "category": "街舞机构 / 少儿舞蹈",
  "image": "/images/cases/studio-uniform.jpg",
  "summary": "适合日常训练、招生展示、校区统一和暑期活动。",
  "craft": "230g 纯棉 T 恤、速干训练服、成人童装混码、长期补货",
  "scenario": "连锁校区、少儿街舞机构、暑期集训、机构统一校服",
  "href": "/clothing-customization"
}
```

字段说明：

- `title`：案例标题
- `category`：客户类型或项目类型
- `image`：案例图片路径
- `summary`：一句话说明这个案例适合什么
- `craft`：工艺、面料、品类重点
- `scenario`：适合的使用场景
- `href`：点击后推荐跳转的服务页

## 2. 怎么替换案例图片

案例图片放在：

```text
src/images/cases/
```

推荐图片格式：

- JPG
- 横图优先，比例接近 `16:9`
- 单张建议控制在 300KB-800KB
- 文件名用英文或拼音，例如 `summer-uniform.jpg`

替换方法：

1. 把新图片放进 `src/images/cases/`
2. 修改 `content/cases.json` 里的 `image`
3. 重新生成网站

例如：

```json
"image": "/images/cases/summer-uniform.jpg"
```

## 3. 怎么重新生成网站

在项目根目录运行：

```bash
python3 scripts/build_site.py
```

生成结果在：

```text
dist/
```

## 4. 怎么本地预览

```bash
cd dist
python3 -m http.server 4173
```

然后打开：

```text
http://127.0.0.1:4173/
```

## 5. 怎么替换线上网站

推荐流程：

1. 把当前项目推到 GitHub 仓库 `keepinggear-svg/zmasterpro-site`
2. 在 Vercel 里把项目连接到这个 GitHub 仓库
3. Vercel 的构建设置使用：
   - Build Command: `python3 scripts/build_site.py`
   - Output Directory: `dist`
4. 在 Vercel 项目里绑定域名：
   - `www.zmasterpro.com`
   - `zmasterpro.com` 可跳转到 `www.zmasterpro.com`

以后只要改 `content/cases.json` 或图片，再推送到 GitHub，Vercel 就会自动重新部署。

## 6. 如果想要真正后台

当前版本是“轻后台”：用 JSON 和图片文件维护内容，简单、稳定、便宜。

如果后续要做网页后台，可以接：

- GitHub 网页编辑：最简单，不需要额外服务器
- Decap CMS：可以做 `/admin` 后台，但需要配置 GitHub OAuth
- Supabase / 企业微信 / 飞书表格：适合后续把表单、客户线索和案例管理做完整
